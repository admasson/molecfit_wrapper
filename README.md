# Molecfit Wrapper Project

This project provides a clean Python workflow to run ESO Molecfit on 1D spectra for:

- ESPRESSO
- HARPS
- NIRPS
- CARMENES (VIS & NIR channels)

It includes:

- a production wrapper script: `Molecfit_parallel_wrapper.py`
- an interface notebook for preparation and visualization: `wrapper_user_guide.ipynb`

Molecfit official page:
https://www.eso.org/sci/software/pipelines/skytools/molecfit

## Project goals

The wrapper is designed to make Molecfit runs reproducible and easier to share by:

- enforcing one consistent input format (`path,berv` CSV)
- mapping instrument-specific FITS headers via explicit configuration
- preparing Molecfit temporary inputs automatically
- collecting outputs in predictable folders and summary tables

## Clean workflow

1. Use `wrapper_user_guide.ipynb` to load data, build S1D files when needed, and create the wrapper CSV.
2. In `Molecfit_parallel_wrapper.py`, set:
	 - `PATH_TO_LIST`
	 - `CONFIGURED_CHANNEL_TYPE`
3. Run:

```bash
python Molecfit_parallel_wrapper.py
```

4. Inspect run folders in `Output/` and consolidated products in `Final_Results_Molecfit/`.

## Data and folder structure

Expected structure in this repository:

- `Molecfit_parallel_wrapper.py`
	- Main executable wrapper.
- `wrapper_user_guide.ipynb`
	- User interface notebook (prepare inputs + inspect results).
- `Parameter_Files/`
	- Static assets:
		- `generic_parfile.par` (template with placeholders)
		- `tellurics_include_*.dat` and `tellurics_exclude_*.dat`
- `Automated_Program/`
	- Runtime inputs and temporary files:
		- CSV file lists (`path,berv`)
		- temporary `.dat` and `.par` files generated during run
		- optional error table if some spectra fail
- `s1d_carmenes_nir/`, `s1d_carmenes_vis/`
	- Example S1D FITS inputs generated by notebook cells.
- `Output/`
	- Per-spectrum Molecfit run directories.
	- Naming pattern: `output{index}_{channel}_{timestamp}`
- `Final_Results_Molecfit/`
	- Consolidated outputs copied by wrapper:
		- one `*_TAC.dat` per processed spectrum
		- one `result_listing_*.csv`
- `backup/`
	- Archived legacy files and old outputs not required in the clean workflow.

## Input CSV format

The wrapper expects a CSV with at least:

- `path`: path to the input FITS file
- `berv`: BERV in m/s (required for channels using `csv_bary_to_topo`)

Example:

```csv
path,berv
./s1d_carmenes_nir/example_obs_S1D_NIR.fits,3500.2
```

## Instrument configuration model

All instrument behavior is centralized in `CHANNEL_CONFIGS` in the wrapper:

- FITS layout (`fits_level`, wavelength/error extraction mode)
- wavelength medium (`vac_air`)
- resolution seed (`resolving_power`)
- BERV policy (`berv_mode`)
- molecules and fit flags (`list_molec`, `fit_molec`, `relcol`)
- include/exclude windows
- header keyword mappings for observing and weather metadata

To add an instrument, copy an existing channel block and adapt all keys explicitly.

## Output products

For each spectrum:

- Molecfit run files in one folder under `Output/`
- corrected TAC table copied to `Final_Results_Molecfit/.../*_TAC.dat`

For each global run:

- summary CSV in `Final_Results_Molecfit/.../result_listing_*.csv`
- optional error CSV in `Automated_Program/` if any failures occur

## Notes for new users

- Use the notebook first: it is the easiest way to build correct inputs.
- Keep paths absolute or repository-relative, but be consistent.
- If you switch instrument, always update both wrapper variables:
	- `PATH_TO_LIST`
	- `CONFIGURED_CHANNEL_TYPE`

## Troubleshooting

- "Missing berv in CSV": add/repair `berv` column or use an instrument config with `berv_mode='none'`.
- "No telluric fitting windows overlap": verify include window files and wavelength units.
- Import warnings for `pandas`/`astropy`: select the correct Python environment and install dependencies.
